<Update label="2024-09-20" description="v1.1.0" tags={["feature", "improvement"]}>

## New Features

- Integrated API documentation components like `<ParamField>` and `<ResponseField>`
- Added support for Mermaid diagrams via code blocks (no JSX component needed)

## Improvements

- Optimized page load times by 40% with lazy-loading images and components
- Improved accessibility with better alt text handling and keyboard navigation

## Bug Fixes

- Corrected parsing errors from unescaped special characters in prose (`{`, `}`, `<`, `>`)

</Update>

<Update label="2024-08-10" description="v1.0.0" tags={["breaking", "feature"]}>

## New Features

- Launched core MDX documentation system with YAML frontmatter and JSX components
- Enabled 15+ components including `<Steps>`, `<Tabs>`, `<Callout>`, and `<CodeGroup>`

## Breaking Changes

- Removed legacy H1 headings; all pages now require H2 as first body heading
- Updated attribute syntax to require double quotes for strings (e.g., `title="Text"`)

## Initial Setup

- Basic authentication via `YOUR_API_KEY`
- Dashboard access at `https://dashboard.example.com`

</Update>

## How to Upgrade

Upgrading Francisco Pina Comendador keeps your documentation space secure and feature-rich. Follow these steps to update smoothly.

<Callout kind="info">
Before upgrading, back up your current configuration and review breaking changes in the latest release notes above.
</Callout>

<Steps>
  <Step title="Check Compatibility" icon="check-circle">

Review your current version against the changelog. Ensure no deprecated components like `<Mermaid>` (use code blocks instead).

  </Step>
  <Step title="Update Package" icon="download">
```bash
npm install francisco-pina-comendador@latest
```
Or use Yarn:
````bash
yarn add francisco-pina-comendador@latest
````
  </Step>
  <Step title="Run Migration" icon="settings">

Execute the migration script:

<CodeGroup tabs="JavaScript,Python">
```javascript
// migration.js
const { migrateDocs } = require('francisco-pina-comendador');
migrateDocs({ from: 'v1.1.0', to: 'v1.2.0' });
```
```python
# migration.py
from francisco_pina_comendador import migrate_docs
migrate_docs(from_version='v1.1.0', to_version='v1.2.0')
```
</CodeGroup>

  </Step>
  <Step title="Test Changes" icon="play-circle">

Verify key pages like `/introduction` and `/quickstart`. Check for parsing errors from unescaped `{` or `<`.

  </Step>
</Steps>
