Focus on content while the platform handles organization and version history.

## Document Creation and Editing

Jump into document creation with an intuitive editor supporting Markdown and MDX.

<Steps>
  <Step title="Create New Document" icon="plus">
    Click the `New Document` button and select a template.
  </Step>
  <Step title="Edit Content" icon="edit">
    Use the WYSIWYG editor or switch to source mode for MDX.
  </Step>
  <Step title="Add Components" icon="components">
    Insert `<Callout>`, `<Tabs>`, or code blocks directly.
  </Step>
</Steps>

```mdx
# My Project Doc

## Introduction

<Callout kind="info">
  This is your starting point.
</Callout>
```

## Version Control for Docs

Maintain document integrity with built-in version control. You commit changes, view diffs, and revert as needed.

<CodeGroup tabs="CLI,API">
  ````bash
  # Commit changes
  fpcd commit -m "Add collaboration section"
  
  # View history
  fpcd log --pretty=oneline
  ````

  ```javascript
  // API commit
  await fetch('https://api.example.com/docs/commit', {
    method: 'POST',
    headers: { 'Authorization': `Bearer ${YOUR_API_KEY}` },
    body: JSON.stringify({
      docId: 'proj-123',
      message: 'Update features'
    })
  });
  ```
</CodeGroup>

<Expandable title="Advanced Branching" default-open="false">
  Create branches for experiments:

  | Action | Command |
  |--------|---------|
  | Create branch | `fpcd branch feature/export` |
  | Switch branch | `fpcd checkout feature/export` |
  | Merge | `fpcd merge feature/export` |
</Expandable>

## Search and Organization Options

Organize docs with tags, folders, and powerful search.

- Use `tags={["api", "guide"]}` for categorization.
- Full-text search supports fuzzy matching and filters.

## Export and Sharing Functionalities

Share your docs widely. Export to multiple formats or generate shareable links.

<Tabs>
